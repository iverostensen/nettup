# Phase 16.1: Remove per-item pricing from wizard, show only total estimate - Context

**Gathered:** 2026-03-06
**Status:** Ready for planning

<domain>
## Phase Boundary

Change the result step of the SmartPrisKalkulator wizard to remove per-item prices from each line item row, showing only the total estimate. The list of selected items remains as a selection summary, but no individual prices are shown next to items. Clipboard, monthly cost, and discount display are also updated to match.

</domain>

<decisions>
## Implementation Decisions

### Line items
- Keep item rows — show selected items as a selection summary without any price column
- Keep category headers (Størrelse, Funksjoner, Integrasjoner, Design) for scannability
- Size tier label alone is sufficient (no short descriptor needed)
- Remove `flex justify-between` layout from item rows — switch to simple left-aligned text list

### Clipboard text
- Clipboard matches the screen: selected items (no prices) + total estimate
- Keep the "Kopier estimat" button — still useful for pasting total + selections into chat/email

### Monthly cost
- Keep the "Drift og hosting" row — it is a fixed recurring cost, not a per-item price, and is important for the full financial picture

### Visual layout
- Total price becomes more prominent: larger font, visually emphasized as the focal point of the result
- Keep the launch discount display: crossed-out original price, "Lanseringstilbud" badge, and "Spar X kr" savings line
- Item rows above the total are styled as secondary/supporting information

### Claude's Discretion
- Exact font size for the more prominent total
- Spacing adjustments after removing price columns from item rows

</decisions>

<code_context>
## Existing Code Insights

### Reusable Assets
- `ResultStep.tsx` (`src/components/islands/wizard/steps/ResultStep.tsx`): The only file that needs changes. Contains all item rendering, total display, discount logic, clipboard, and monthly cost sections.
- `fadeUp`, `staggerContainer`, `springs` from `@/lib/animation`: already used in ResultStep, no changes needed
- `calculateEstimate` from `@/lib/calculate-estimate`: returns `lineItems`, `discounted`, `oneTime`, `monthly`, `discountActive`, `discountPercent` — all still needed except `formatItemPrice` and `getSizeRange` can be removed

### Established Patterns
- Item rows currently use `flex justify-between` with a label span on the left and a price span on the right — remove the price span and change to plain `<span>` or `<li>` list
- `grouped` array (items grouped by category) can stay — just render label only, no price
- `buildClipboardText()` function needs updating: remove per-item price lines, keep item names + total

### Integration Points
- `SmartPrisKalkulator.tsx` passes `state` and `onReset` to `ResultStep` — no changes needed there
- No changes needed to `calculateEstimate`, `pricing-config`, or any other wizard step

</code_context>
