<feature>
  <name>Additive Calculation Engine</name>
  <files>src/lib/calculate-estimate.ts, src/lib/__tests__/calculate-estimate.test.ts</files>
  <behavior>
    - Base estimate (no add-ons): returns size tier min/max as oneTime range, empty line items except size
    - Adding features: each feature's fixed price added to both min and max
    - Adding integrations: same additive behavior as features
    - Design level: Standard (price=0) adds nothing, Skreddersydd/Premium adds their price to both min/max
    - Discount active: discounted.min = Math.round(oneTime.min * 0.6), discounted.max = Math.round(oneTime.max * 0.6)
    - Discount inactive: discounted equals oneTime (no reduction)
    - Line items: one per selected item (size always present; design only if price > 0; each feature; each integration)
    - Monthly: returns the service's flat monthlyPrice unchanged (no discount applied)
    - Error cases: unknown sizeId throws, unknown featureId throws, unknown integrationId throws, unknown designId throws
    - All 3 service types work: nettside, nettbutikk, landingsside
  </behavior>
  <implementation>
Install Vitest as dev dependency. Add "test" and "test:watch" scripts to package.json. Write tests first (RED), then implement calculate-estimate.ts (GREEN).

The engine function `calculateEstimate(request: EstimateRequest): EstimateResult`:
1. Look up service config from pricingConfig.services[request.serviceType]
2. Find and validate size tier by request.sizeId -- throw if not found
3. Build lineItems array: add size (using minPrice for line item display)
4. For each featureId: find in service.features, throw if missing, push to lineItems
5. For each integrationId: find in service.integrations, throw if missing, push to lineItems
6. Find and validate design by request.designId -- throw if not found. Push to lineItems only if price > 0
7. Calculate oneTime: min = size.minPrice + sum of all add-on prices, max = size.maxPrice + sum of all add-on prices
8. Calculate discounted: if discount.active, multiply oneTime by (1 - discount.percentage) and Math.round; else same as oneTime
9. Return full EstimateResult with lineItems, oneTime, discounted, monthly, discountPercent (percentage * 100), discountActive

Import pricingConfig and all types from `@/config/pricing-config`. Do NOT modify pricing-config.ts.
  </implementation>
</feature>
