The `<Button href="/kontakt" size="lg">Ta kontakt</Button>` renders a server-side anchor. Replace it with a plain `<a>` that carries a `data-city` attribute and has an `id` for the inline script to attach to.

Replace the `<div class="mt-8">` block in the CTA section with:
```astro
<div class="mt-8">
  <a
    href="/kontakt"
    id="city-cta"
    data-city={city.slug}
    class="inline-flex items-center justify-center rounded-md bg-brand px-6 py-3 text-base font-semibold text-surface transition-colors hover:bg-brand-light focus:outline-none focus:ring-2 focus:ring-brand focus:ring-offset-2 focus:ring-offset-surface"
  >
    Ta kontakt
  </a>
</div>
```

Then, at the bottom of the file (before `</BaseLayout>`), add the tracking script:
```astro
<script is:inline>
  (function () {
    var btn = document.getElementById('city-cta');
    if (btn) {
      btn.addEventListener('click', function () {
        var city = btn.getAttribute('data-city');
        if (city && window.plausible) {
          window.plausible('City CTA Clicked', { props: { city: city } });
        }
      });
    }
  })();
</script>
```

Use `is:inline` so the script is not bundled/tree-shaken and runs immediately in the browser. The IIFE wrapper avoids polluting the global scope. Note: this script uses `window.plausible` directly (not importing analytics.ts) because `is:inline` scripts cannot import ES modules.

Do NOT import `trackCityCtaClicked` from analytics.ts here — that function is for React islands only. The inline script approach is the correct Astro pattern for non-island pages.

Verify `city.slug` is in scope by checking that the frontmatter already destructures it from `getStaticPaths` (it should be — this is how the page builds its content). Read the top of the file if uncertain.
