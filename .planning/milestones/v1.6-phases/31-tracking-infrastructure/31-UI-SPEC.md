## Copywriting Contract

All content in Norwegian (bokmal).

| Element | Copy |
|---------|------|
| Banner text | "Vi bruker informasjonskapsler for a male annonseeffekt. Les mer i var [personvernerklaering](/personvern)." |
| Accept button | "Godta" |
| Decline button | "Avsla" |
| Privacy page - Consent Mode section heading | "Informasjonskapsler og samtykke" |
| Privacy page - What accepting means | Plain-language explanation: accepting allows Google to store cookies for measuring ad effectiveness. Declining means Google still loads but sends anonymous, cookieless data -- no personal tracking. |
| Privacy page - Plausible contrast | Plausible Analytics runs without cookies and requires no consent. It collects no personal data. This is separate from Google's tracking. |

### Consent Banner Interaction Contract

| State | Behavior |
|-------|----------|
| First visit (no stored choice) | Banner visible at bottom of viewport, fixed position |
| Returning visitor (stored choice) | Banner never shown |
| Accept clicked | Banner fades out (opacity 0) + slides down (translateY 100%) over 300ms, then removed from DOM |
| Decline clicked | Same dismiss animation as accept |
| Banner position | Fixed bottom, full width, z-50 |
| Banner background | `bg-surface-raised/95 backdrop-blur-md` with `border-t border-white/10` |

### Privacy Page Content Contract

The /personvern page must be updated to include a new section covering Consent Mode v2. The section must:

1. Explain in plain Norwegian what cookies the site uses and why (Google Ads measurement)
2. Explain what "Godta" means: Google stores cookies to track ad conversions
3. Explain what "Avsla" means: Google still loads but sends only anonymous data, no cookies stored
4. Explain the two new consent types (`ad_user_data`, `ad_personalization`) in non-technical language -- describe what data is shared, not the parameter names
5. Contrast with Plausible: runs completely without cookies, no consent needed
6. Reference the localStorage key name for technical transparency

No CTAs, no empty states, no error states -- this phase has no user-facing forms or data displays.

## Phase-Specific UI Notes

### What changes visually

1. **Consent banner text** -- adds inline link to /personvern (currently just "Vi bruker cookies for a male annonseeffekt." with no link)
2. **Privacy page content** -- new section explaining Consent Mode v2 behavior in plain Norwegian

### What does NOT change visually

- Banner position, animation, button styles, button labels -- all stay exactly as they are
- No new components created
- No layout changes
- No typography changes
- No color changes
- No spacing changes

### Inline link styling in banner

The /personvern link inside the banner text must use:
- Color: `text-brand` (#06b6d4)
- Hover: `hover:text-brand-light` (#22d3ee)
- Underline: `underline underline-offset-2`
- Transition: `transition-colors duration-150`

This matches the existing anchor styling pattern in the prose-article class.
