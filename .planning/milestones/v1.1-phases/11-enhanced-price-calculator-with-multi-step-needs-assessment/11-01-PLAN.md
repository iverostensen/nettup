Complete rewrite of PrisKalkulatorIsland.tsx. The 11-RESEARCH.md contains full content specification, data shapes, JSX patterns, and pitfall warnings. Key changes:

**Types and state:**
- Change `Phase` from `'pick' | 'narrow' | 'result'` to `'goal' | 'recommend' | 'narrow' | 'result'`
- Keep same `State` interface, just update `phase` default to `'goal'`
- `initialState.phase` = `'goal'` (was `'pick'`)

**Data structures (all hardcoded in component):**
- Add `GoalOption` interface with `label`, `subLabel`, `service: ServiceSlug` fields
- Add `goalOptions` array with 3 entries (exact copy from RESEARCH.md "Goal Options" section)
- Add `includedItems: Record<ServiceSlug, string[]>` (exact copy from RESEARCH.md "Included Items Data Shape")
- Expand `narrowingQuestions` to 2 questions per service:
  - Q1 options: `priceEstimate: ''` (empty — Q1 does not determine price)
  - Q2 options: carry the final `priceEstimate` values (from RESEARCH.md "Narrowing Questions" section)
  - Use proper Norwegian characters (a with ring, o with stroke) not ASCII approximations

**Handlers:**
- Replace `handlePickService` with `handleGoalSelect(option: GoalOption)` — sets `selectedService` to `option.service`, advances to `'recommend'`
- Add `handleRecommendContinue()` — advances to `'narrow'` with `narrowStep: 0`
- `handleNarrowOption` logic stays the same (already handles multi-question arrays — stores price only on last question)
- `handleReset` resets to `initialState` (which now uses `'goal'` phase)

**Animation key:**
- Update `animKey` to cover all 4 phases: `'goal'`, `'recommend'`, `'narrow-N'`, `'result'`

**JSX phases (all use existing slideVariants + springs.gentle pattern):**

1. **Goal phase** — replaces old `pick` phase entirely:
   - Intro text above buttons: "Svar pa 2-3 sporsmal og fa et prisestimat — tar under ett minutt." (use proper Norwegian chars)
   - Question heading: "Hva er malet ditt?" (proper chars)
   - 3 goal option buttons (same button styling as current service picker), each showing `label` + `subLabel`

2. **Recommend phase** — new phase:
   - Small caps brand-colored label: "Vi anbefaler"
   - Service name (large, bold) + tagline (muted)
   - Reassurance text: "Basert pa malet ditt er dette det beste utgangspunktet. La oss finne riktig pris." (proper chars)
   - Single button: "Se sporsmaalene ->" (outline style, same as option buttons but centered text)

3. **Narrow phase** — same structure, now shows "Sporsmal 1 av 2" / "Sporsmal 2 av 2":
   - Step counter already works (`narrowStep + 1` of `totalQuestions`)
   - Q1 buttons: show label only (no price since `priceEstimate` is empty)
   - Q2 buttons: show label + price estimate (existing pattern)
   - Conditionally render price: only show `<span>` for `option.priceEstimate` when it is non-empty

4. **Result phase** — enhanced:
   - Keep existing: brand label, service name, tagline, price estimate, monthly note
   - Add included items list between price and CTA: `<ul>` with checkmark SVG + text per item (use camelCase SVG attributes for JSX)
   - Add "Les mer om {name.toLowerCase()}" text link in the CTA row (after "Start pa nytt" button)
   - CTA row: "Kom i gang" button + "Start pa nytt" button + "Les mer" link

**Important pitfalls to avoid (from RESEARCH.md):**
- Use unique animKey for each phase (especially 'goal' vs 'recommend')
- Q1 options MUST have `priceEstimate: ''` — only Q2 carries real prices
- Delete ALL old `pick` phase code — no remnants
- Use `.toLowerCase()` on service name in "Les mer" link
- Use camelCase for SVG attributes in JSX (strokeWidth, strokeLinecap, strokeLinejoin)
- Use proper Norwegian characters throughout (not ASCII approximations from research)
