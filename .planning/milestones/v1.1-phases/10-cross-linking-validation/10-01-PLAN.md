Add a "Relaterte tjenester" section to all 7 service sub-pages, positioned between the FAQ and CTA sections. Each page will show 2 cross-link cards pointing to related services, using data from `services.ts` as the single source of truth.

<task type="auto" wave="2">
  <name>Task 2: Create shared RelaterteTjenester.astro component</name>
  <files>src/components/sections/RelaterteTjenester.astro</files>
  <action>
Create `src/components/sections/RelaterteTjenester.astro`. This is a shared component used by all 7 service pages.

Interface:
```ts
interface Props {
  related: string[];  // slugs from services.ts
}
```

Implementation:
1. Import `services` from `@/config/services`, `Card` from `@/components/ui/Card.astro`, `Section` from `@/components/ui/Section.astro`, `SectionHeader` from `@/components/ui/SectionHeader.astro`.
2. Receive `related` prop. Resolve full service objects: `related.map(slug => services.find(s => s.slug === slug)).filter(Boolean)`.
3. Guard: only render section when `relatedServices.length > 0`.
4. Section wrapper: `<Section background="default" padding="md">`.
5. Section header: `<SectionHeader title="Relaterte tjenester" centered={true} />`.
6. Cards grid: `<div class="grid grid-cols-1 gap-6 sm:grid-cols-2 lg:grid-cols-3">` — with max 2 cards this produces a 2-column row on sm+.
7. Each card: `<Card as="a" href={`/tjenester/${service.slug}`} hover padding="lg" class="flex flex-col">` containing:
   - `<h3 class="mb-1 font-semibold text-text">{service.name}</h3>`
   - `<p class="mb-3 text-sm text-brand">{service.tagline}</p>`
   - `<p class="mt-auto text-sm font-semibold text-text-muted">{service.priceRange}</p>`

Do NOT nest `<a>` inside `<a>`. The Card renders as `<a>` when `href` is set, so all inner content must be non-anchor elements only (h3, p, span).
