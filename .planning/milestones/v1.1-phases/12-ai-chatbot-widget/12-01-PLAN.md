<task type="auto">
  <name>Task 2: Create system prompt builder and streaming API endpoint</name>
  <files>src/config/chatbot.ts, src/pages/api/chat.ts</files>
  <action>
1. Create `src/config/chatbot.ts`:
   - Export `buildSystemPrompt(currentPage: string): string`
   - Import `services` from `./services`
   - Build service info block from services array: name, tagline, priceRange (use launchPriceRange since launch discount is active), monthlyPriceLabel if present
   - System prompt personality: Norwegian (bokmal), professional but approachable, transparently AI ("Jeg er Nettup sin AI-assistent"), quotes price RANGES only (never exact), falls back to /kontakt when it can't answer
   - Include page-context function that maps currentPage to relevant context string:
     - `/tjenester/[slug]` -> focuses on that specific service
     - `/tjenester` -> user is exploring all services
     - `/kontakt` -> user is already on contact page
     - `/` -> user is on homepage
     - Other pages -> generic context
   - Keep system prompt under ~800 tokens. Include service data, behavioral rules, page context. No FAQ content dump (token bloat pitfall from research).

2. Create `src/pages/api/chat.ts`:
   - `export const prerender = false;` at top (critical — this makes it a serverless function)
   - Import `Anthropic` from `@anthropic-ai/sdk`
   - Import `buildSystemPrompt` from `@/config/chatbot`
   - Create Anthropic client with `import.meta.env.ANTHROPIC_API_KEY`
   - Export `POST: APIRoute` handler:
     - Parse request body: `{ messages: Array<{role: 'user'|'assistant', content: string}>, currentPage: string }`
     - Validate messages array exists and is non-empty, return 400 if invalid
     - Add basic rate-limit guard: limit messages array to max 20 entries (trim from front if longer)
     - Call `client.messages.stream()` with model `claude-haiku-4-5-20241022`, max_tokens 1024, system prompt from buildSystemPrompt(currentPage)
     - Create ReadableStream that:
       - On `text` event: enqueue `data: {"text":"..."}\n\n`
       - On `end` event: enqueue `data: [DONE]\n\n` and close
       - On `error` event: enqueue error JSON and close
     - Return Response with headers: Content-Type text/event-stream, Cache-Control no-cache, Connection keep-alive
     - Wrap entire handler in try/catch — return 500 JSON on unexpected errors

Use TypeScript strict mode. No `any` types. Use `APIRoute` type from `astro`.
  </action>
