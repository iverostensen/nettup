In tjenester/index.astro, import `pakker` from `@/config/pricing` and compute the Service schemas in the frontmatter. Then inject them into the page using BaseLayout's `<slot name="head">` if one exists, or by placing them in a `<head>` slot.

BaseLayout does not have a named head slot. The cleanest approach for a single page is to add the schema blocks directly in the page markup, placed as the first element inside `<main>` (Astro will not complain about script tags inside main, and they will be rendered). However, the correct SEO placement is in `<head>`.

Use this approach: Add a named `<slot name="head" />` to BaseLayout.astro `<head>` section (place it just before the closing `</head>` tag). Then in tjenester/index.astro, pass schemas via `<Fragment slot="head">`.

IMPORTANT check: If BaseLayout already has a `<slot name="head">` — just use it. If not, add it.

In tjenester/index.astro frontmatter, add:

```typescript
import { pakker } from '@/config/pricing';

const serviceSchemas = pakker.map(pakke => ({
  "@context": "https://schema.org",
  "@type": "Service",
  "name": `Nettup ${pakke.name}`,
  "description": pakke.description,
  "provider": {
    "@type": "Organization",
    "name": "Nettup",
    "url": "https://nettup.no"
  },
  "areaServed": {
    "@type": "Country",
    "name": "Norway"
  },
  "offers": {
    "@type": "Offer",
    "price": pakke.launchPrice.replace(/\s/g, ''),
    "priceCurrency": "NOK"
  }
}));
```

In the BaseLayout call, pass schemas via the head slot:

```astro
<BaseLayout title="..." description="...">
  <Fragment slot="head">
    {serviceSchemas.map(schema => (
      <script type="application/ld+json" set:html={JSON.stringify(schema)} />
    ))}
  </Fragment>
  <main>
    ...
  </main>
</BaseLayout>
```

And in BaseLayout.astro `<head>`, add just before `</head>`:

```astro
    <slot name="head" />
  </head>
```
