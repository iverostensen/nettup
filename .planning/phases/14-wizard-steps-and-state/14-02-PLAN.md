Create all 5 step components. Each step receives dispatch and relevant state/data via props. Norwegian headings in conversational question format.

**GoalStep.tsx:**
- Props: `onSelectGoal: (serviceType: ServiceType) => void`
- Heading: "Hva er malet ditt?"
- Subtitle: "Svar pa noen sporsmal og fa et prisestimat -- tar under ett minutt."
- Renders 3 GoalCard components with these options:
  - nettside: "Fa flere kunder til bedriften" / "Profesjonell nettside som konverterer besokende"
  - nettbutikk: "Selge produkter pa nett" / "Komplett nettbutikk med betaling og lagerstyring"
  - landingsside: "Markedsfore en kampanje eller tilbud" / "En fokusert landingsside som overbeviser"
- NOTE: Use proper Norwegian characters (a with ring, o with slash) -- the ASCII above is for plan readability only

**SizeStep.tsx:**
- Props: `serviceType: ServiceType`, `onSelectSize: (sizeId: string) => void`
- Heading: "Hvor stor er nettsiden?" (adapt wording per service type -- "Hvor mange produkter?" for nettbutikk, "Hvor stor er landingssiden?" for landingsside)
- Reads sizes from `pricingConfig.services[serviceType].sizes`
- Renders SelectableCard for each size with price formatted as range "8 000 - 12 000 kr" using `toLocaleString('nb-NO')`
- Single-select: clicking a card calls onSelectSize (auto-advances via reducer)
- Cards show as unselected (no persistent selection highlight needed since auto-advance)

**FeaturesStep.tsx:**
- Props: `serviceType: ServiceType`, `selectedIds: string[]`, `onToggle: (featureId: string) => void`, `onNext: () => void`
- Heading: "Hvilke funksjoner trenger du?"
- Reads features from `pricingConfig.services[serviceType].features`
- 2-column grid on desktop (`grid grid-cols-1 md:grid-cols-2 gap-3`), 1-column on mobile
- Renders SelectableCard for each feature with price formatted as "+ 3 000 kr"
- Multi-select: cards toggle on/off, selected cards show check icon
- "Neste" button always visible below the grid (even with 0 selections -- features are optional)
- "Neste" button styled: `w-full rounded-md bg-brand py-3 text-center font-semibold text-white hover:bg-brand-light transition-colors`

**IntegrationsStep.tsx:**
- Same pattern as FeaturesStep but reads integrations from config
- Heading: "Trenger du integrasjoner?"
- Same multi-select behavior with Neste button

**DesignStep.tsx:**
- Props: `serviceType: ServiceType`, `onSelectDesign: (designId: string) => void`
- Heading: "Hvilket designniva onsker du?"
- Reads designs from `pricingConfig.services[serviceType].designs`
- Renders SelectableCard for each design level. Standard shows "Inkludert" instead of "+ 0 kr" when price is 0. Others show "+ 5 000 kr" format.
- Single-select: auto-advances on click (same as SizeStep)

**SmartPrisKalkulator.tsx:**
- Default export: `SmartPrisKalkulator`
- Uses `useReducer(wizardReducer, initialWizardState)` for state
- Uses `useReducedMotion()` from framer-motion for accessibility
- Container: `mx-auto max-w-3xl` (wider than old wizard per research recommendation)
- Renders WizardStepper at top with currentStepIndex computed from `STEP_ORDER.indexOf(state.currentStep)` and onStepClick dispatching GO_TO_STEP
- Below stepper: "Tilbake" button (text style, with left arrow) visible on all steps except goal. On click dispatches GO_TO_STEP to previous step.
- Step content wrapped in `AnimatePresence mode="wait" custom={state.direction}`:
  - motion.div with key={state.currentStep}, custom={state.direction}
  - slideVariants with direction-aware x offset (40px, or 0 if prefersReducedMotion)
  - transition={springs.gentle}
- Conditional rendering based on state.currentStep:
  - 'goal': GoalStep
  - 'size': SizeStep (only if serviceType selected)
  - 'features': FeaturesStep
  - 'integrations': IntegrationsStep
  - 'design': DesignStep
  - 'result': placeholder div with text "Resultat kommer i fase 15" (Phase 15 builds result display)
- Step content inside a container div with `rounded-md border border-white/10 bg-surface-raised p-6 sm:p-8`
- Dispatch mapping: GoalStep.onSelectGoal -> SELECT_GOAL, SizeStep.onSelectSize -> SELECT_SIZE, etc.
