Create /Users/iverostensen/nettup/src/components/islands/HeroIsland.tsx.

**Layout:** Two-column grid on desktop (lg:grid-cols-2), single column centered on mobile. Left = text content. Right = stats card (desktop only, lg:flex).

**Right-side element (Claude's discretion):** A floating metrics/stats card showing 3-4 Nettup performance claims (load time, Lighthouse score, delivery time). This communicates technical capability with numbers rather than an image. Animates in with springPop + delay.

**Animation architecture:**
- Wrap LEFT column in `<motion.div>` with `variants={heroContainer}` as the stagger orchestrator
- Each child text element gets `variants={fadeUp}` and `transition={springs.snappy}` or `springs.gentle`
- Right stats card gets `variants={springPop}` with `transition={{ ...springs.gentle, delay: 0.35 }}`
- `initial="hidden" animate="visible"` on orchestrating containers (NOT whileInView — hero plays on mount)
- Full `useReducedMotion` guard: when true, pass `{}` as variants and skip animation props entirely

**Hero text content (Norwegian):**
- H1: "Nettsider som" + `<RotatingText />` on the rotating word line
- Subtitle: "Vi lager moderne, raske nettsider for norske bedrifter. Uten stress, til rett pris."
- CTA row: "Ta kontakt" (primary) + "Se vårt arbeid" (secondary) linking to /kontakt and /prosjekter

**Stats card content (right side, desktop):**
```
⚡ < 1s lastetid
100 Lighthouse score
2 uker levering
✓ Fastpris
```

**Key rules:**
- Every animated element must be `motion.div`, `motion.h1`, `motion.p` etc. — NOT plain `<div>` (variants won't flow)
- `viewport={{ once: true }}` is NOT used — hero uses `animate` not `whileInView`
- Do NOT add `transition:persist` anywhere in this component
- `useReducedMotion()` from framer-motion (not a custom hook)
- The section itself keeps the dark background (`bg-surface`) + `grain-overlay` class + bottom gradient fade
- Remove the video element — new hero does not use video background
- Keep the scroll indicator at the bottom (adapt as motion.div with fadeIn variant, delay ~0.8s)

**Reduced motion path:** When `shouldReduceMotion` is true, render all content statically (no motion.* wrappers needed — or use motion.div with empty variants `{}` so content appears immediately without transition).

**Imports:**
