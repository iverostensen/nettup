**Step 1 — Add legacy collections flag to astro.config.mjs:**
Add `legacy: { collections: true }` to the defineConfig object. This is required for Astro 5 to recognize `src/content/config.ts` (the legacy path). Without it, `getCollection('blogg')` returns `[]` silently.

```javascript
export default defineConfig({
  site: 'https://nettup.no',
  output: 'static',
  adapter: vercel(),
  integrations: [react(), sitemap()],
  vite: { plugins: [tailwindcss()] },
  legacy: {
    collections: true,
  },
});
```

**Step 2 — Create src/content/config.ts (new file):**
Use the legacy Content Collections API. Schema fields:
- `title` — `z.string()` — conversational H1
- `seoTitle` — `z.string()` — keyword-first, goes in `<title>` only
- `category` — `z.string()` — free string, no enum
- `date` — `z.coerce.date()` — coerce YAML date strings
- `readTime` — `z.number().int().positive()` — minutes as integer
- `description` — `z.string()` — meta description + og:description
- `relatedSlugs` — `z.array(z.string()).optional()` — array of slugs
- `faq` — `z.array(z.object({ question: z.string(), answer: z.string() })).optional()` — for FAQPage JSON-LD

```typescript
import { defineCollection, z } from 'astro:content';

const blogg = defineCollection({
  type: 'content',
  schema: z.object({
    title: z.string(),
    seoTitle: z.string(),
    category: z.string(),
    date: z.coerce.date(),
    readTime: z.number().int().positive(),
    description: z.string(),
    relatedSlugs: z.array(z.string()).optional(),
    faq: z.array(z.object({
      question: z.string(),
      answer: z.string(),
    })).optional(),
  }),
});

export const collections = { blogg };
```

**RelatedArticles.astro:**
Fetches all articles via `getCollection('blogg')`, filters to the `relatedSlugs` list, excludes the current article by slug, limits to 3.

```typescript
import { getCollection } from 'astro:content';
import type { CollectionEntry } from 'astro:content';
import ArticleCard from './ArticleCard.astro';

interface Props {
  relatedSlugs?: string[];
  currentSlug: string;
}

const { relatedSlugs = [], currentSlug } = Astro.props;

const allArticles = await getCollection('blogg');
const related: CollectionEntry<'blogg'>[] = relatedSlugs
  .map(slug => allArticles.find(a => a.slug === slug))
  .filter((a): a is NonNullable<typeof a> => a !== undefined)
  .filter(a => a.slug !== currentSlug)
  .slice(0, 3);
```

Only render the section if `related.length > 0`. Heading: `Les også` as `h2` with `text-xl font-semibold mb-6`. Render `ArticleCard` for each in a responsive grid (1 col mobile, 2-3 cols on md+, matching the count of articles).

Do NOT render the section at all (not even the heading) when `related` is empty — this prevents orphaned "Les også" headings.
