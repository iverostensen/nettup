  <name>Task 2: discover-topics.ts — Stage 1 topic selection + queue management</name>
  <files>scripts/blog/discover-topics.ts</files>
  <action>
Create `scripts/blog/discover-topics.ts` implementing `selectTopic()` and queue I/O helpers.

**Queue file path:** Resolve from `import.meta.url` so it works from any cwd:
```typescript
import { fileURLToPath } from 'url';
import path from 'path';
const __dir = path.dirname(fileURLToPath(import.meta.url));
const QUEUE_PATH = path.join(__dir, 'topics-queue.json');
const BLOGG_DIR = path.resolve(__dir, '../../src/content/blogg');
```

**readQueue / writeQueue helpers:**
- `readQueue()`: return `[]` if QUEUE_PATH doesn't exist (fs.existsSync check). Parse JSON otherwise.
- `writeQueue(entries)`: JSON.stringify with 2-space indent, write to QUEUE_PATH.

**readExistingSlugs helper:**
- If BLOGG_DIR doesn't exist (`fs.existsSync` check), return `[]`.
- Otherwise `fs.readdirSync(BLOGG_DIR)` and strip `.md` extension from filenames.

**selectTopic() logic:**
1. Load queue via `readQueue()`.
2. Load existing slugs via `readExistingSlugs()`.
3. Find retryable entry: a queue entry with status `'rejected'` and `attempts < RETRY_LIMIT`. Prefer the one with fewest attempts. If found, increment `attempts`, set status back to `'pending'`, write queue, return the entry (with `previousFailureReason` populated from entry's `reason`).
4. If no retryable entry: select a new cluster using round-robin weighted by remaining capacity. Remaining capacity = `cluster.articles_target - countPublished(cluster)` where `countPublished` counts entries in queue with status `'published'` for that cluster. Pick the cluster with most remaining capacity (if tied, pick the one appearing first in CLUSTERS).
5. Ask Claude to pick a specific article angle: use a SHORT Anthropic API call (max_tokens: 512) to get a `{ title, slug }` JSON object. Prompt: list cluster keywords + existing slugs (so Claude avoids overlap). Use `norwegianSlugify` on the returned slug to ensure correct format.
6. If the generated slug already exists in `readExistingSlugs()`, retry with a different angle (up to 3 attempts, then throw an error with message "Could not find unique topic after 3 attempts").
7. Write a new `QueueEntry` with `status: 'pending'`, `attempts: 0`, `createdAt: new Date().toISOString()` to queue. Return the entry.

**Exports:** `selectTopic`, `readQueue`, `writeQueue`, `readExistingSlugs`
