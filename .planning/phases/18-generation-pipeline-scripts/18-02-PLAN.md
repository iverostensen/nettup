**Inline lixScore function** (no external dependency):
```typescript
function lixScore(text: string): number {
  const sentences = text.split(/[.!?]+/).filter(s => s.trim().length > 0);
  const words = text.split(/\s+/).filter(w => w.trim().length > 0);
  const longWords = words.filter(w => w.replace(/[^a-zA-ZæøåÆØÅ]/g, '').length > 6);
  if (sentences.length === 0 || words.length === 0) return 0;
  return (words.length / sentences.length) + (longWords.length * 100 / words.length);
}
```

**LIX threshold: 55 (NOT 45).** The architecture doc says 45 but that is wrong for Norwegian — see STATE.md and CONTEXT.md. Use 55.

**Pass 1 — Claude self-review:**
Use the review system prompt from the architecture doc Stage 3 verbatim. Send the article body as user message. Parse the response to extract 6 scores (1-10 each). The model should return scores inline — parse integers from the response using regex (look for `\d+` after each criterion name). Compute average. If average < 7, set `passed = false`, `reason = 'AI review: average score ' + avg + '/10'`.

Pass 1 Claude call: `max_tokens: 512`, `CLAUDE_MODEL` from config.

**Pass 2 — Automated checks** (only run if pass 1 passed, to avoid wasting time on already-rejected articles):
- `wordCount`: count words by splitting on `\s+`, check >= 1500
- `lixScore`: call `lixScore(article.markdownBody)` <= 55
- `hasFaqSection`: `/##\s*Vanlige spørsmål/i.test(article.markdownBody)`
- `nettupMentions`: count occurrences of `Nettup` in body <= 2

If any automated check fails, set `passed = false`, `reason` = the specific failing check + value.

**On rejection:** Update the queue entry — increment `attempts`, if `attempts >= RETRY_LIMIT` set status `'permanently_rejected'`, else set status `'rejected'` with `reason`. Write updated queue.

**runQualityGate(article: ArticleResult): Promise&lt;QualityResult&gt;** — orchestrates both passes, handles queue update on rejection.

**writeJobSummary helper** (writes to $GITHUB_STEP_SUMMARY when in GitHub Actions):
```typescript
import fs from 'node:fs';

function writeJobSummary(content: string): void {
  if (process.env.GITHUB_ACTIONS && process.env.GITHUB_STEP_SUMMARY) {
    fs.appendFileSync(process.env.GITHUB_STEP_SUMMARY, content + '\n');
  }
}
```

**Structured progress printing** — console.log with stage labels:

Use a format function like `log(stage: string, msg: string)` that pads the stage label.

**Main pipeline function `runPipeline()`:**

1. `[ Topic ]` — call `selectTopic()`. Log selected topic title and cluster. If selectTopic throws, log reason and writeJobSummary, then return (do not throw).

2. `[ Generate ]` — call `generateArticle(topic)`. Log word count of markdownBody. If generateArticle throws (API error, JSON parse failure), log error, writeJobSummary, update queue entry to `'rejected'` with reason, return.

3. `[ Quality ]` — call `runQualityGate(article)`. Log scores. If `!quality.passed`: log rejection reason, writeJobSummary with rejection summary, return. Queue update happens inside `runQualityGate` already.

4. `[ PR ]` — call `publishArticle(article, quality)`. Log PR URL. writeJobSummary with success summary (PR URL + quality scores).

**Entry point — wrap in try/catch, always exit 0:**
```typescript
async function main(): Promise<void> {
  try {
    await runPipeline();
  } catch (err) {
    const message = err instanceof Error ? err.message : String(err);
    console.error('[ Error ]  Unexpected pipeline failure:', message);
    writeJobSummary('## Pipeline Error\n\n' + message);
  }
  process.exit(0);  // Always exit 0 — rejection is expected behavior, not CI failure
}

main();
```
