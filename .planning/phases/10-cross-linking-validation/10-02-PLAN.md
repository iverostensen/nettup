<task type="checkpoint:human-verify" wave="2">
  <name>Task 2: Validate JSON-LD schemas via Google Rich Results Test</name>
  <files></files>
  <action>
Manual validation of Service and FAQPage JSON-LD schemas on all 7 service sub-pages.

**How to test (use HTML paste method — works before deployment):**

For each of the 7 pages, do the following:
1. Run `npm run build && npm run preview` to serve the production build locally at `http://localhost:4321`
2. Open each service page URL in your browser, view page source, and copy the full HTML
3. Go to https://search.google.com/test/rich-results
4. Click "Code snippet" tab and paste the HTML
5. Run the test

**URLs to test:**
- http://localhost:4321/tjenester/nettside/
- http://localhost:4321/tjenester/nettbutikk/
- http://localhost:4321/tjenester/landingsside/
- http://localhost:4321/tjenester/webapp/
- http://localhost:4321/tjenester/seo/
- http://localhost:4321/tjenester/ai/
- http://localhost:4321/tjenester/vedlikehold/

**What to look for — pass criteria:**
- Each page shows detected "Service" rich result with no errors
- Each page shows detected "FAQ" rich result with no errors
- No warnings about missing required fields on either schema type

**Common issues to watch for:**
- Duplicate FAQPage schemas (would mean FAQ.astro is rendered twice) — should not occur with current pattern
- Missing `priceCurrency` field — already in schema, should pass
- Empty `mainEntity` array in FAQPage — would mean FAQ questions aren't populating the schema

If any page fails validation, note which page and which error, and report back before marking complete.
  </action>
