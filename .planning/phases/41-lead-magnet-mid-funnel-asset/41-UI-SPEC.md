## Component Inventory

### Page Structure

```
/sjekkliste/index.astro
  BaseLayout
    Section (padding="lg", background="default")
      hero area: Display heading + subheading
    Section (background="raised", rounded)
      SjekklisterIsland.tsx (React island, client:visible)
        ChecklistItem x10 (items 1-3 visible, 4-10 blurred)
        GateOverlay (lock icon + form)
        PostSubmitCTA (revealed after unlock)
```

### SjekklisterIsland States

| State | What Renders |
|-------|-------------|
| `locked` | Items 1-3 fully visible. Items 4-10 with `blur(8px)` CSS filter + lock icon overlay. 2-field form (navn, e-post) centered over blurred area. |
| `submitting` | Form button shows spinner + "Sender..." text. Inputs disabled. |
| `error` | Red error banner below form: problem description + retry button. |
| `unlocked` | Blur removed with staggered fade-in animation. All 10 items visible. CTA card fades in below checklist. |

### Checklist Item Layout

Each item is a horizontal row:

```
[Number badge]  [Title]
                [Description - 1-2 sentences]
```

- Number badge: 32x32px circle, `bg-brand/10 text-brand font-semibold text-sm` for visible items, `bg-white/5 text-text-muted` for locked items
- Title: `text-base font-semibold text-text`
- Description: `text-sm text-text-muted` at line-height 1.5
- Vertical gap between items: 16px (`gap-4`)

### Gate Form

- Positioned as an overlay card centered over items 4-10
- Card: `bg-surface-raised border border-white/10 rounded-xl p-6 md:p-8`
- Lock icon: 24x24px SVG, `text-brand`, centered above form heading
- Form heading: `text-lg font-semibold text-text` -- "Fa hele sjekklisten"
- Form subtext: `text-sm text-text-muted` -- "Skriv inn navn og e-post sa sender vi den til deg."
- Fields stacked vertically with 12px gap
- Input styling matches HeroMicroForm: `rounded-lg border border-white/10 bg-surface px-4 py-3 text-text placeholder:text-text-muted/50 focus:border-brand focus:ring-1 focus:ring-brand`
- Submit button: full width, `rounded-lg bg-brand px-6 py-3 font-medium text-surface hover:bg-brand-light`
- Honeypot field: hidden `_gotcha` input (project spam protection convention)

### Post-Submit CTA Card

- Appears below the full checklist after unlock
- Card: `bg-surface-raised border border-white/10 rounded-xl p-6 md:p-8 text-center`
- Heading: "Vil du at vi fikser dette for deg?"
- Body: "Vi bygger moderne nettsider fra 399 kr/mnd. Null oppstart, ingen bindingstid."
- CTA button: `bg-brand text-surface rounded-lg px-6 py-3 font-medium hover:bg-brand-light`
- Links to `/kontakt?tjeneste=nettside`

## Interaction Contract

### Blur-to-Reveal Animation (Framer Motion)

| Property | Value |
|----------|-------|
| Blur filter on locked items | `blur(8px)` via CSS `filter` |
| Reveal trigger | Successful Formspree POST response (200 OK) |
| Blur removal | `filter: blur(8px)` to `filter: blur(0px)` over 400ms, `ease-out` |
| Item stagger | Each item (4-10) fades in with 80ms stagger delay |
| Item animation | `opacity: 0 -> 1` + `translateY: 12px -> 0` over 400ms |
| CTA card entrance | Fades in 200ms after last item, `opacity: 0 -> 1` over 300ms |
| Reduced motion | Skip all animation, reveal items instantly (`prefers-reduced-motion: reduce`) |

### Form Interactions

| Interaction | Behavior |
|-------------|----------|
| Focus input | Border transitions to `border-brand` + `ring-1 ring-brand` over 150ms |
| Submit | Button shows spinner SVG + "Sender...", both inputs `disabled` |
| Success | No redirect -- blur removes inline, events fire |
| Error | Red text below the failing field, or network error banner below form |
| Retry | "Prov igjen" link resets form to idle state |

### Analytics Events (on successful submit)

1. `trackLeadMagnetDownload()` via `src/lib/analytics.ts` (Plausible custom event)
2. `fbq('track', 'Lead', { content_name: 'Sjekkliste 2026' })` -- consent-gated: `localStorage.getItem('nettup_ads_consent') === 'granted' && window.fbq`

## Accessibility

| Requirement | Implementation |
|-------------|---------------|
| Focus visibility | 2px solid `#06b6d4` outline, 2px offset (global CSS) |
| Form labels | Visible `<label>` elements for both name and email fields |
| Error announcements | `role="alert"` + `aria-live="assertive"` on error messages |
| Invalid state | `aria-invalid="true"` on fields with validation errors |
| Reduced motion | `prefers-reduced-motion: reduce` disables all Framer Motion animations, items reveal instantly |
| Semantic HTML | `<form>`, `<ol>` for checklist, `<h1>` for display heading |
| Blur accessibility | Blurred items have `aria-hidden="true"` to prevent screen readers from reading locked content |
