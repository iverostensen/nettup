  <action>
Create `.github/workflows/blog-generate.yml` with the following exact structure:

```yaml
name: Blog Generate

on:
  schedule:
    - cron: '0 8 * * 1'  # Every Monday at 08:00 UTC
  workflow_dispatch:       # Manual trigger — no inputs needed

concurrency:
  group: blog-generate
  cancel-in-progress: true

jobs:
  generate:
    name: Generate Blog Article
    runs-on: ubuntu-latest

    steps:
      - name: Checkout
        uses: actions/checkout@v4
        with:
          token: ${{ secrets.GH_PAT }}  # PAT required — GITHUB_TOKEN cannot trigger CI on its own PRs

      - name: Setup Node.js
        uses: actions/setup-node@v4
        with:
          node-version: '20'
          cache: 'npm'

      - name: Install dependencies
        run: npm ci

      - name: Validate secrets
        run: |
          if [ -z "$ANTHROPIC_API_KEY" ]; then
            echo "## Pipeline Error" >> $GITHUB_STEP_SUMMARY
            echo "" >> $GITHUB_STEP_SUMMARY
            echo "ANTHROPIC_API_KEY secret is not set. Add it in Settings → Secrets and variables → Actions." >> $GITHUB_STEP_SUMMARY
            exit 0
          fi
          if [ -z "$GH_PAT" ]; then
            echo "## Pipeline Error" >> $GITHUB_STEP_SUMMARY
            echo "" >> $GITHUB_STEP_SUMMARY
            echo "GH_PAT secret is not set. Add a PAT with contents:write and pull-requests:write scopes." >> $GITHUB_STEP_SUMMARY
            exit 0
          fi
        env:
          ANTHROPIC_API_KEY: ${{ secrets.ANTHROPIC_API_KEY }}
          GH_PAT: ${{ secrets.GH_PAT }}

      - name: Run blog pipeline
        run: npx tsx scripts/blog/index.ts
        env:
          ANTHROPIC_API_KEY: ${{ secrets.ANTHROPIC_API_KEY }}
          GITHUB_TOKEN: ${{ secrets.GH_PAT }}  # Exposed as GITHUB_TOKEN so Octokit in publish.ts picks it up
          GITHUB_REPOSITORY: ${{ github.repository }}

      - name: Enable auto-merge on blog PR
        run: |
          PR=$(gh pr list --head "blogg/" --json number --jq '.[0].number' 2>/dev/null || echo "")
          if [ -n "$PR" ]; then
            gh pr merge "$PR" --auto --squash
            echo "Auto-merge enabled on PR #$PR" >> $GITHUB_STEP_SUMMARY
          fi
        env:
          GH_TOKEN: ${{ secrets.GH_PAT }}
        continue-on-error: true
```

Key design decisions:
- PAT used for checkout so pushed `blogg/*` branches trigger CI (GITHUB_TOKEN cannot trigger CI on PRs it creates — GitHub loop-prevention)
- Concurrency group prevents simultaneous generation runs
- Secret validation step writes job summary and exits 0 — never sends CI failure email (CI-03)
- GITHUB_TOKEN env var set to GH_PAT value so Octokit in publish.ts authenticates with the PAT
- GITHUB_REPOSITORY passed explicitly so publish.ts resolveRepoIdentifier() uses env var path (avoids git remote parsing)
- Auto-merge step uses `continue-on-error: true` — safe if no PR was created (rejection path) or PR already merged

Configure repository settings so auto-merge works correctly when CI passes on blog PRs.

**Step 1: Derive the repo slug**
```bash
git remote get-url origin
# Expected output: git@github.com:iverostensen/nettup.git (or https variant)
# Repo slug = iverostensen/nettup
```

**Step 2: Enable auto-merge at repo level**
```bash
gh repo edit --enable-auto-merge
```

**Step 3: Set branch protection rule on main**

The rule must require the `Lint & Build` status check (exact job name from ci.yml) and must NOT require PR reviews (a review requirement would block auto-merge on unattended PRs).

```bash
REPO=$(git remote get-url origin | sed 's/.*github\.com[:/]\(.*\)\.git/\1/')

gh api "repos/${REPO}/branches/main/protection" \
  --method PUT \
  --input - <<'EOF'
{
  "required_status_checks": {
    "strict": false,
    "contexts": ["Lint & Build"]
  },
  "enforce_admins": false,
  "required_pull_request_reviews": null,
  "restrictions": null
}
EOF
```

Note: If `required_pull_request_reviews: null` causes an API error, replace it with:
```json
"required_pull_request_reviews": {
  "required_approving_review_count": 0
}
```

**Step 4: Verify the rule is active**
```bash
REPO=$(git remote get-url origin | sed 's/.*github\.com[:/]\(.*\)\.git/\1/')
gh api "repos/${REPO}/branches/main/protection" --jq '.required_status_checks.contexts'
```
Expected output: `["Lint & Build"]`

**Step 5: Verify auto-merge is enabled**
```bash
gh repo view --json autoMergeAllowed --jq .autoMergeAllowed
```
Expected output: `true`
