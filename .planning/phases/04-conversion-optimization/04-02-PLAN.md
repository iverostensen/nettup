<!-- Audit checklist from research — use exactly this. -->

Mobile audit criteria (375px DevTools viewport):
  - No horizontal scrollbar (check body overflow-x)
  - All <a> and <button> elements at least 44px tall (min-h-11 in Tailwind = 44px)
  - Pricing grid on /tjenester: grid-cols-3 must have responsive guard (md:grid-cols-3 or similar)
  - Contact form fields on /kontakt: padding adequate for thumb interaction
  - FloatingNav items: tap area covers at least 44px vertically
  - Footer links: adequate tap area
  - Images: don't overflow their containers

Tailwind fix patterns:
  - Tap target: add `min-h-11` to the <a> or <button> element, pair with `flex items-center`
  - Overflow: ensure no fixed `px` width wider than 375px; prefer `w-full` or `max-w-*` with overflow-hidden
  - Grid collapse: `grid-cols-1 md:grid-cols-3` (not bare `grid-cols-3`)

Common candidates (check these first):
  - FloatingNav.tsx: nav link anchor elements
  - Footer.astro: footer link elements
  - Pakker.astro: grid already uses md:grid-cols-3 — verify it starts at 1 col on mobile
  - ContactForm.tsx: input padding classes

Scope: Fix what you find. If a check passes, note it and move on. This is not a redesign.
</interfaces>
