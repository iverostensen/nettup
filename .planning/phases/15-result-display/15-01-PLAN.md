**Core logic:**
- Call `calculateEstimate()` with `{ serviceType: state.serviceType!, sizeId: state.sizeId!, featureIds: state.featureIds, integrationIds: state.integrationIds, designId: state.designId! }` (non-null assertions safe because SmartPrisKalkulator guards nulls before rendering)
- Group `lineItems` by category using defined order: `['size', 'feature', 'integration', 'design']`
- Skip empty categories (if no items in a category, do not render that group heading)

**Category labels (Norwegian):**
```typescript
const CATEGORY_LABELS: Record<LineItem['category'], string> = {
  size: 'Storrelse',
  feature: 'Funksjoner',
  integration: 'Integrasjoner',
  design: 'Design',
};
```

**Size line item special handling (CRITICAL -- Pitfall 1):**
For `category === 'size'`, look up the full SizeTier from `pricingConfig.services[estimate.serviceType].sizes.find(s => s.id === item.id)` to get both `minPrice` and `maxPrice`. Display as range: "1-5 sider: 8 000 - 12 000 kr". Other categories show fixed price: "CMS: 3 000 kr".

**Price formatting:** Use `price.toLocaleString('nb-NO')` consistently for Norwegian number formatting (space as thousands separator).

**Discount presentation (Pitfall 2):**
- If `discountActive`: Show "Lanseringstilbud" badge (small pill with bg-brand/10 text-brand), then the total section shows:
  - Crossed-out original `oneTime` range (text-text-muted line-through)
  - Discounted range in brand color (text-brand, font-bold, larger text)
  - Savings line: "Spar X 000 - Y 000 kr" in green (text-emerald-400)
- If NOT `discountActive`: Show `oneTime` range cleanly -- no badge, no strikethrough, no savings line

**Monthly cost section:**
- Separate from one-time total, with a border-t divider
- Label: "Drift og hosting"
- Show `monthly` value formatted with "kr/mnd" suffix
- NOT subject to discount (Pitfall 3) -- display the raw `monthly` value as returned by engine

**Disclaimer:**
- Muted text below everything: "Dette er et estimat -- endelig pris avhenger av prosjektets omfang."

**Three action buttons:**
1. Primary: "Kontakt oss for tilbud" -- `<a>` tag linking to `/kontakt?tjeneste=${serviceType}&estimat=${discounted.min}-${discounted.max}`. Same tab (no target="_blank"). Style: bg-brand text-surface font-semibold rounded-md px-6 py-3 hover:bg-brand-light transition-colors
2. Secondary: "Beregn pa nytt" -- calls `onReset()`. Style: border border-white/20 text-text rounded-md px-6 py-3 hover:border-white/40 transition-colors
3. Tertiary: "Kopier estimat" -- copies plain text to clipboard via `navigator.clipboard.writeText()`. Shows a clipboard icon that changes to a checkmark for 2 seconds on success. Wrap in try/catch (Pitfall 5). Style: text-text-muted text-sm flex items-center gap-1.5 hover:text-text transition-colors

- Each line item row: flex justify-between with label left and price right
